Product-minded software engineer who doesn't just write code — I analyze products, find what's missing, and build it.

Currently interning at **Larinsoft**, where I independently audited an enterprise messaging platform, proposed 14 feature improvements, designed an on-premise AI architecture, and am now implementing it. ## What I do

**I find gaps in products and fill them with engineering.**

At Larinsoft, I was given access to CladChat (enterprise messaging app) and PhotoFace (event photo matching platform). Nobody asked me to audit them — I did it because I saw things that could be better. That audit turned into a technical report, the report turned into an architecture, and the architecture is now becoming real code.

## Current work

- **CladKnowledge** — Designing and implementing an on-premise AI chatbot for enterprise messaging. Ollama + Qwen2.5, RAG pipeline with ChromaDB, fully air-gapped (no data leaves the server). Modules: chat summarization, task extraction, document Q&A, smart reply, expert routing.

- **PhotoFace** — Built an event-based face recognition platform end-to-end. Benchmarked InsightFace against the existing FaceRecognitionDotNet model → proved superior accuracy → migrated company's production system. Architecture: .NET API, Angular admin, Flutter mobile, FastAPI AI service, PostgreSQL + MinIO.

- **CladChat Product Audit** — Full-stack analysis of a closed-system enterprise messaging app (Flutter/BLoC, .NET, SignalR, Jitsi). Benchmarked against Teams, Slack, Rocket.Chat. Delivered 14 feature proposals covering security, enterprise compliance, and AI integration.

## Graduation thesis

**Facial Expression Recognition** — ConvNeXt-Tiny on RAF-DB with WeightedRandomSampler, class-weighted loss, and mixup augmentation. Two-stage fine-tuning, MTCNN face detection, Unity socket integration. Each dataset switch (FER2013 → FERPlus → RAF-DB) was driven by a real failure I identified and solved.
